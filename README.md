# FMCG_DE_Project
End-to-End Data Engineering project on Databricks

🚀 Completed an End-to-End Data Engineering project on Databricks, and yes it involved AI Agents.

🏭 The Business Problem (FMCG Domain):
I simulated a real-world industry scenario where a parent company had acquired a smaller startup. The challenge was to consolidate messy, fragmented data from CSV files and cloud drives into a single, reliable analytics platform, apply transformations and make it BI ready.

⚙️ I built an end-to-end ETL pipeline on Databricks, covering:
‣ Ingestion
‣ Data Modeling using Star Schema.
‣Transforming data across medallion layers (Bronze 🥉→ Silver 🥈→ Gold🥇), thus ensuring clean, reliable, analytics-ready data at every layer
‣ Historical + Incremental data loads built for both legacy systems and fast-growing startup data.
‣ Resolved real schema mismatches between legacy ERP and startup data sources.
‣ Built scalable pipelines that don't break when data volume grows
‣ leveraged AI-driven 🤖 natural language querying on top of the Gold layer, making this an AI-ready pipeline
‣ Used Databricks Dashboard 📊 to visualize key revenue and sales trends for business stakeholders using Databricks Genie 🤖.

🛠️ Tech Stack:
Python · SQL · PySpark · Delta Lake · Medallion Architecture · Databricks Genie (AI Agents)

💡 Key Learnings:

‣ The biggest takeaway? The "grunt work" of data engineering like cleaning, deduplication, schema resolution are the true backbone of any successful AI or Analytics initiative. No AI agent can save you from bad data.
‣ Experienced firsthand how Databricks unifies data engineering, analytics, and AI in a single platform.
‣ This project reinforced that the future of Data Engineering isn't just about moving data, it's about building intelligent pipelines powered by AI.
